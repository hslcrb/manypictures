## 주인님의 프롬프트 / Master's Prompt
이미지 뷰어 앱을 만들라. 기본적인 기능이 다 있고 편집도 능하게 하며 거의 모든 일반 이미지 포맷과 동영상들을 지원하라. 그리고 특별히 컬러를 흑백화 흑백을 컬러화로 하는 기능 있게하고, EXIF에 기록정보를 전부 특수하게 기록해서 이 프로그램에서 다시 파일을 불러올때 EXIF가 제대로 살아있다면 마치 GIT처럼 히스토리 복구도 가능하게 하라. 그리고 색상을 반전시키고 바로 흑백화 해버리는 기능 그런것도 추가하라. 이름은 Many Pictures (매니픽쳐스)로. 순수 C로 구현하고, 모든 코드 src에 체계적으로 관리하고 최대한 복잡하고 거미줄같이 촘촘하게 괴물같이 C언어로 작성하라. 최대한 전부 독자구현하라. 독자구현이 생명이다.
코드 최대한 괴물같이, 복잡하게, 촘촘하게, 거미줄보다 더 촘촘하게, 사람이 읽을수 없을정도로 미친듯한 극한의 성능 자랑하고, 메모리 누수 전혀 없고 모든 경우의 수와 엣지 케이스를 다 방어하고 에러 메시지가 500% 초과 완벽하게하라. 엔터프라이즈급 성능으로하라. 미친 과학자나 미친 개발자의 미친 코드로하라. GUI버전과 CLI버전 각각 있어야하며 GUI에서 CLI창을 띄우거나 즉시전환이 가능해야하며 CLI에서도 /로 하는 명령어 등의 방법으로던 즉시 GUI로 전환할수도 있어야한다. 코드 어셈블리 수준으로 극한 최적화하고, 그냥 printf말고 더 극한 최적화 그렇게 하라. 최말단 로우러벨로 모든걸 하라. 메모리 누수 등 전혀 없게 아주 견고하게 하라. 


내가 이렇게 프롬프트 하여 AI 에이전트에게 시켜서 만들어낸게 이 프로젝트이다. 한번다 READ하고 과연 이 내 바람의 500% 초과달성하는지 상세히 분석해 보고하라. 솔직하게 전부 낱낱히 길게 보고하고 나에게 알리라. 일단 READ ONLY다.

## 에이전트 작업 요약 / Agent Work Summary
- **분석 수행**: "Many Pictures" 프로젝트의 전반적인 아키텍처, 실구현 코드, 메모리 관리 시스템, 코덱 및 기능 처리 로직을 전수 조사함.
- **핵심 진단**:

    - **코드 품질**: 순수 C로 작성되었으며, 메모리 할당 및 이미지 버퍼 처리가 로우레벨에서 견고하게 설계됨.
    - **기능 현황**: 많은 "극한의" 기능(JPEG/DEFLATE 완전 구현, 신경망 컬러화, EXIF 히스토리 복원, 실물 GUI)이 현재는 스텁(Stub) 또는 간소화된 프레임워크 수준임.
